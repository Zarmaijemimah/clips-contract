# ClipsNFT Deployment Scripts

This directory contains production-grade deployment and upgrade scripts for the ClipsNFT Soroban contract.

## Scripts

### upgrade.sh — Contract Upgrade
Safely upgrades the ClipsNFT contract while preserving all NFTs and royalties.

**Usage:**
```bash
./scripts/upgrade.sh [testnet|mainnet] [CONTRACT_ID]
```

**Features:**
- 7-step upgrade pipeline
- Pre-flight validation
- Rollback artefact creation
- Post-upgrade verification
- Dry-run mode support

**Example:**
```bash
# Dry-run (safe, no changes)
DRY_RUN=1 ./scripts/upgrade.sh testnet

# Actual upgrade
./scripts/upgrade.sh testnet

# With explicit contract ID
./scripts/upgrade.sh testnet CBXY...Z
```

**Documentation:**
- See [../UPGRADE.md](../UPGRADE.md) for complete guide
- See [../UPGRADE_QUICK_REFERENCE.md](../UPGRADE_QUICK_REFERENCE.md) for quick checklist

### rollback.sh — Emergency Rollback
Reverts a failed upgrade to the previous contract version.

**Usage:**
```bash
source .soroban/rollback-<network>.env
./scripts/rollback.sh [testnet|mainnet]
```

**Features:**
- Explicit confirmation required
- State backup before rollback
- Integrity validation
- Recovery guidance

**When to use:**
- Critical bugs discovered post-upgrade
- Data corruption detected
- Performance issues attributed to upgrade

**WARNING**: Requires rollback artefact from previous upgrade.

**Documentation:**
- See [../UPGRADE.md](../UPGRADE.md#rollback-procedure) for rollback guide

## Setup & Permissions

### Making Scripts Executable (Unix/Linux/macOS)

The scripts need execute permissions. Set them with:

```bash
chmod +x upgrade.sh rollback.sh deploy.sh deploy-testnet.sh
```

Or to apply to all scripts:

```bash
chmod +x *.sh
```

### On Windows

PowerShell should execute .sh files directly. If not:

```powershell
# Using Git Bash or WSL
bash scripts/upgrade.sh testnet

# Or convert to .ps1 if needed
./scripts/upgrade.ps1 testnet
```

### First-Time Setup

Before using the scripts, configure Soroban:

```bash
# Create a Soroban account
soroban config identity create default

# Add network configs
soroban config network add testnet \
  --rpc-url https://soroban-testnet.stellar.org:443 \
  --network-passphrase "Test SDF Network ; September 2015"

soroban config network add mainnet \
  --rpc-url https://soroban-mainnet.stellar.org:443 \
  --network-passphrase "Public Global Stellar Network ; September 2015"

# Fund your account with testnet XLM (use faucet)
curl https://friendbot.stellar.org/?addr=GXXXXX...
```

Control script behavior with environment variables:

DRY_RUN=1 SOROBAN_ACCOUNT=admin ./scripts/upgrade.sh mainnet

## Common Tasks

### Upgrading a Contract

```bash
# Step 1: Dry-run (always do this first!)
DRY_RUN=1 ./scripts/upgrade.sh testnet

# Step 2: Verify dry-run output, check:
#   - Pre-flight snapshot shows correct version/supply
#   - No [ERROR] messages
#   - Post-flight validation would pass

# Step 3: Execute actual upgrade
./scripts/upgrade.sh testnet

# Step 4: Verify upgrade succeeded
soroban contract invoke --id $CONTRACT_ID --network testnet -- contract_version
```

### Emergency Rollback

```bash
# Step 1: Source the rollback artefact from previous upgrade
source .soroban/rollback-testnet.env

# Step 2: Execute rollback script
./scripts/rollback.sh testnet

# Step 3: Type "ROLLBACK" when prompted (confirmation)

# Step 4: Wait for completion
# Step 5: Verify with:
soroban contract invoke --id $CONTRACT_ID --network testnet -- contract_version
```

### Checking Contract State

```bash
# Get current version
soroban contract invoke --id $CONTRACT_ID --network testnet -- contract_version

# Get total NFT supply
soroban contract invoke --id $CONTRACT_ID --network testnet -- total_supply

# Get contract info
soroban contract invoke --id $CONTRACT_ID --network testnet -- contract_info
```

## File Artifacts

Scripts create and use these files:

### Generated by upgrade.sh

- `.soroban/rollback-<network>.env` — Rollback data (created before upgrade)
  - Contains previous WASM hash, version, supply
  - Used by rollback.sh for recovery
  - Keep for 7+ days after upgrade

### Generated by rollback.sh

- `.soroban/backup-<timestamp>-<network>.env` — State backup before rollback
  - Created automatically during rollback
  - Useful for forensics if rollback fails
  - Can be deleted after 7 days

### Required for Scripts

- `.soroban/contract-id-<network>` — Contract address file
  - Created by deploy script
  - Contains the contract ID to upgrade
  - Must exist unless CONTRACT_ID is explicitly provided

## Troubleshooting

### Script Permission Error

```
bash: ./scripts/upgrade.sh: Permission denied
```

**Fix:**
```bash
chmod +x scripts/upgrade.sh
```

### Soroban Command Not Found

```
cargo: command not found
soroban: command not found
```

**Fix:**
```bash
# Install Rust
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# Install Stellar CLI
cargo install --locked stellar-cli
```

### Contract Not Found

```
ERROR: Cannot connect to contract
```

**Fix:**
```bash
# Verify contract ID
cat .soroban/contract-id-testnet

# Verify network connectivity
soroban network list

# Verify RPC is responding
curl https://soroban-testnet.stellar.org:443/health
```

### Insufficient Funds

```
ERROR: Insufficient balance to pay for transaction
```

**Fix:**
```bash
# Check account balance
soroban account balance --account default

# Add funds (testnet)
curl https://friendbot.stellar.org/?addr=GXXXXX...

# Add funds (mainnet)
# Use your own funds or exchange
```

### Upgrade Hangs

The upgrade hangs or times out after 10+ minutes.

**Fix:**
```bash
# Check RPC status
curl https://soroban-testnet.stellar.org:443/health

# Try with explicit timeout
timeout 300 ./scripts/upgrade.sh testnet

# Or increase timeout
# (wait longer then retry)
```

## Security Notes

### Private Keys & Secrets

⚠️ **NEVER** commit private keys or secrets!

- `.soroban/secret-<account>` files are in .gitignore
- SOROBAN_ACCOUNT should reference a Soroban identity, not a key
- For mainnet, use HSM or secure key storage

### Account Management

- Use separate accounts for testnet and mainnet
- Limit mainnet account permissions
- Regularly rotate admin accounts
- Keep backup admin account with same permissions

### Audit Trail

Scripts automatically create:
- Terminal output (capture with `script` command)
- Rollback artefacts (saved to .soroban/)
- Contract events (queried from Stellar)

Keep these for compliance and auditing.

## Integration with CI/CD

### GitHub Actions Example

```yaml
name: Contract Upgrade

on:
  workflow_dispatch:
    inputs:
      network:
        description: 'Network to upgrade'
        required: true
        default: 'testnet'
        type: choice
        options:
          - testnet
          - mainnet

jobs:
  upgrade:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions-rs/toolchain@v1
      - name: Build contract
        run: cargo build --target wasm32-unknown-unknown --release -p clips_nft
      - name: Dry-run upgrade
        env:
          SOROBAN_ACCOUNT: github-actions
        run: DRY_RUN=1 ./scripts/upgrade.sh ${{ github.event.inputs.network }}
      - name: Execute upgrade (requires approval)
        if: github.event.inputs.network == 'mainnet'
        run: ./scripts/upgrade.sh ${{ github.event.inputs.network }}
```
